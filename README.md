# codebase-explorer-mcp

An MCP server that analyzes any codebase — detects project type, languages, framework, entry points, API routes, controllers, services, components, tests, and more.

## Prerequisites

- **Python 3.11+**
- **`uv`** — a fast Python package manager. [`mcp dev`](#test-with-mcp-inspector-development) uses `uvx` under the hood, so `uv` must be on PATH.

  Install `uv`:
  ```powershell
  # Windows
  powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

  # macOS / Linux
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```

  Verify with `uv --version`.

## Setup

```bash
# 1. Sync dependencies (reads uv.lock, creates .venv, installs exact versions)
uv sync

# 2. Run the server
uv run server.py
```

> **Why `uv` and `uv.lock`?**
>
> `uv.lock` is a lockfile automatically generated by `uv` (like `package-lock.json` in npm or `Cargo.lock` in Rust). It records the **exact** version and content hash of every dependency, ensuring everyone on the team gets **identical environments** — no more "works on my machine" issues.
>
> `mcp dev` (the MCP Inspector) relies on `uvx` under the hood, which requires `uv` to be installed. Without it, you'd see:
> ```
> 'uv' is not recognized as an internal or external command
> ```
>
> Installing `uv` and running `uv sync` solves this by making `uv` available on PATH and generating `uv.lock` to pin all dependency versions.

### Alternative (without `uv`)

```bash
python -m venv venv
venv\Scripts\activate     # Windows
# source venv/bin/activate  # macOS / Linux
pip install "mcp[cli]>=1.6.0"
python server.py
```

## Test with MCP Inspector (development)

The MCP Inspector opens a web UI to test each tool interactively.

```bash
# From the project folder (uv must be on PATH):
mcp dev server.py
```

If `mcp` is not on PATH, use the full path:
```bash
.venv\Scripts\mcp.exe dev server.py
```

Open the URL shown in the terminal (usually `http://localhost:6274`). You'll see all 5 tools listed — click one, fill in the args, and test it live.

## Usage with Claude Desktop

Make sure your venv's Python is used. Find its path:

```bash
# Windows (with venv active):
where python
# Example output: C:\path\to\codebase-explorer-mcp\venv\Scripts\python.exe
```

Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "codebase-explorer": {
      "command": "C:\\full\\path\\to\\codebase-explorer-mcp\\venv\\Scripts\\python.exe",
      "args": ["server.py"],
      "cwd": "C:\\full\\path\\to\\codebase-explorer-mcp"
    }
  }
}
```

Restart Claude Desktop. Then ask:

- _"Analyze the structure of my project at C:\path\to\project"_
- _"Find all API routes and controllers in my codebase"_
- _"Search for 'class.*Service' in .py files"_
- _"What features exist in this project?"_

## Tools

| Tool | Required params | Description |
|---|---|---|---|
| `analyze_structure` | `path` | Overview — languages, files, entry points, directory tree |
| `find_features` | `path` | Detect routes, controllers, components, tests, etc. |
| `search_symbols` | `path`, `pattern` | Regex search across files with optional extension filter |
| `get_feature_detail` | `path`, `name` | Show code context around a feature by its name (from `find_features` output) |
| `list_project_types` | _(none)_ | All detectable project types and frameworks |

## Supported Languages

Python, JavaScript/TypeScript, Java, Go, Ruby, C#, Rust, PHP, Kotlin, Swift, and more.
